Traditional Web Applications

  • Client (web browser) sends a request
  • Server receives the request, and sends back an HTML response
  • Most of the work is done at the server
  • Client's role is limited to showing the UI, and capturing inputs

Javascript Evolution

  1. Initially treated as a highly forgiving scripting language
  2. Treated as second class citizen among programming languages
  3. Used primarily for client-side validations
  4. Apart from validation, other features remained undiscovered

AngularJS

  1. Open-source JS framework for Client-side MVC
  2. Community maintained by Google
  3. Community has 30,000 members, and counting


                   http://angularjs.org/

Features

  1. Two-way data binding
  2. #-based routes
  3. Directives
  4. Validation
  5. Dependency injection
  6. Highly testable
  
 Lots of third party modules plug in with AngularJS.

Benefits of angularjs

  1. Uses the computing power & resources of Client Workstation
  2. Reduces session load on the web server
  3. Interaction between Client and Web Server are on need basis
  4. Light-weight application that is quick and responsive

Limitations of AngularJS

  • AngularJS needs additional efforts to work on IE8 - this can pose problems to public-facing sites
  • Initial learning curve
    • Need knowledge on Browser events
    • Terminology - Directives, Services
    • JavaScript as a programming language
  • Structuring and Modularization are challenging
